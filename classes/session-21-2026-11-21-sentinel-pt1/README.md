## About This Session

Microsoft Sentinel is a cloud-native Security Information and Event Management (SIEM) and Security Orchestration, Automation, and Response (SOAR) platform. In simple terms: it collects logs from all your systems, analyses them to find threats, and can automatically respond to incidents.

Part 1 introduces what a SIEM is and why enterprises need one, how to set up a Sentinel workspace, and how to connect data sources (called Data Connectors) — pulling in logs from Microsoft 365, Azure, and third-party tools like firewalls and endpoint agents.

**By the end of this session you will be able to:**
- Explain what SIEM and SOAR mean and why Sentinel is used
- Create and configure a Microsoft Sentinel workspace
- Connect data sources using built-in Data Connectors
- Understand how log ingestion costs work in Sentinel
