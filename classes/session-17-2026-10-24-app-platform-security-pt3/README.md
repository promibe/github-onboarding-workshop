## About This Session

The final app security session focuses on protecting web applications from internet-based attacks. Azure Web Application Firewall (WAF) sits in front of your application and blocks common attacks like SQL injection, cross-site scripting (XSS), and bot traffic before they ever reach your code.

You will also look at Azure Front Door and Application Gateway as delivery platforms for WAF, and how to tune WAF rules so you block attacks without blocking legitimate users.

**By the end of this session you will be able to:**
- Explain what a Web Application Firewall does and when to use one
- Deploy and configure Azure WAF on Application Gateway or Front Door
- Interpret WAF logs to identify attack patterns
- Tune WAF rules to reduce false positives
