## About This Session

Virtual Machines (VMs) are the workhorses of cloud computing — and they are also a major attack surface if not properly hardened. This session starts a three-part series on securing servers and VMs in Azure.

Part 1 covers the basics: enabling Microsoft Defender for Servers, configuring Just-in-Time (JIT) VM access (so RDP and SSH ports are only opened for a specific person for a limited time), applying security baselines, and understanding what Azure Security Benchmark means for compute resources.

**By the end of this session you will be able to:**
- Enable and interpret Microsoft Defender for Servers recommendations
- Configure Just-in-Time VM access to eliminate standing RDP/SSH exposure
- Apply security baselines to harden Windows and Linux VMs
- Explain the Azure Security Benchmark for compute resources
