## About This Session

AI systems introduce new types of security risk that traditional tools were not designed for — prompt injection attacks, data poisoning, model theft, and unintended data exposure through AI outputs.

Part 1 lays the foundation: Microsoft's Responsible AI principles, the AI threat landscape, and how Azure AI services (like Azure OpenAI) are designed with security controls built in. You will look at how to identify where AI risks appear in a system and begin applying security thinking to AI-powered applications.

**By the end of this session you will be able to:**
- Explain the unique security risks that AI workloads introduce
- Describe Microsoft's Responsible AI framework
- Identify how prompt injection and data poisoning attacks work
- Recognise where Azure AI security controls apply
