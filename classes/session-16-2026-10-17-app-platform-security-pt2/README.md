## About This Session

Security has to be built into the software development process, not added on at the end. Part 2 looks at DevSecOps — integrating security checks into CI/CD pipelines — and supply chain security, which is about making sure the code libraries and tools your application depends on have not been tampered with.

You will cover Microsoft Defender for DevOps (scanning code repos and pipelines for secrets and vulnerabilities), GitHub Advanced Security features, and how to review and harden infrastructure-as-code templates before they are deployed.

**By the end of this session you will be able to:**
- Explain what DevSecOps means and why it matters
- Use Microsoft Defender for DevOps to scan pipelines and repositories
- Identify hardcoded secrets and insecure IaC configurations
- Apply supply chain security principles to your software projects
