## About This Session

Governance in cloud security means making sure your organisation follows the rules — both its own internal policies and external regulations like GDPR, ISO 27001, and industry standards.

Part 1 covers the foundational tools Microsoft provides: Azure Policy (automatically enforcing configuration rules across your subscriptions), Management Groups (organising multiple subscriptions), and Microsoft Purview Compliance Manager (tracking your compliance score against regulatory frameworks).

**By the end of this session you will be able to:**
- Explain what cloud governance means and why it matters for SC-500
- Create and assign Azure Policies to enforce security configurations
- Understand how Management Groups help apply rules at scale
- Use Microsoft Purview Compliance Manager to assess your compliance posture
