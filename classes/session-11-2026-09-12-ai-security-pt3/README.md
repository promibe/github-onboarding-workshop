## About This Session

The final AI security session focuses on governance and response. As AI becomes embedded in business processes, organisations need clear policies about how it is used, audited, and corrected when things go wrong.

You will cover Microsoft Purview's AI governance features, how to conduct a security review of an AI system, incident response for AI-related security events, and how the SC-500 exam expects you to reason about AI risk in enterprise scenarios.

**By the end of this session you will be able to:**
- Describe how Microsoft Purview governs AI data usage
- Conduct a basic security assessment of an AI-powered application
- Outline an incident response plan for an AI security event
- Tie AI security concepts to SC-500 exam objectives
